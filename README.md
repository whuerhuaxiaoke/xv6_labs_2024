## 本代码仓库为武汉大学2023级网安“操作系统实验（网安）”课程期末实验的代码仓库

---

具体贡献为：将 xv6 的进程调度路径从原有的两段式切换（进程A → 调度器线程 → 进程B）优化为一次直接切换（进程A → 进程B），从而减少不必要的上下文切换开销，提升调度效率。
实现了：
- 参考scheduler，实现schedule函数，在即将释放CPU的进程中调用该函数，实现进程的切换；
- 创建idle进程，防止无进程可调度运行的情况；
- 通过中断控制和锁顺序，保障并发安全与正确性，防止死锁发生
- 进程调度策略采用多级反馈队列/动态优先级调度
- 用显式的就绪队代替隐式进程链表.

### 同时，为验证优化后的进程调度，编写了一个用户态测试函数```schedtest```

---

## 注意：该进程调度仅适用于单核场景。必须在编译时设置CPU=1，或在Makefile中修改CPU设置。
多核编译会出现```panic: acquire```的情况。
